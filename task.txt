สร้างระบบออกตั๋วซัพพอร์ต (Ticketing System) ภาษาไทยแบบครบถ้วน  
ใช้ฟอนต์ "Anuphan" ทั้งระบบ  
ชื่อ: Tozei Ticketing
logo: png placeholder 1024x512
"Ticketing" after logo
ธีมโทน Light-Blue (พื้นหลังฟ้าอ่อน / ปุ่มฟ้าน้ำเงิน / เส้นและ UI สีฟ้าเข้ม)  
ดีไซน์แบบเรียบ สะอาด ตา โปรเฟสชันแนล

คุณสมบัติที่ต้องมี:
1. ระบบผู้ใช้
   - สมัครสมาชิก
   - ล็อกอิน / ล็อกเอาต์
   - แก้ไขโปรไฟล์
   - ตั้งค่า Dark/Light (ลำดับความสำคัญ Light-Blue)

2. ระบบตั๋ว (Ticket)
   - เปิดตั๋วใหม่ (หัวข้อ, รายละเอียด, หมวดหมู่)
   - อัปโหลดไฟล์แนบแบบ Local Storage (ไม่ใช้ S3/Cloud)
        • รองรับ JPG/PNG/PDF/ZIP และอื่นๆ
        • จำกัดขนาด เช่น ≤ 10MB  
        • บันทึกไว้ใน /uploads หรือ /storage/uploads  
   - ดูสถานะตั๋วของฉัน
   - ตอบกลับในตั๋ว (thread-style)
   - แจ้งเตือนเมื่อมีผู้ดูแลตอบกลับ

3. ฝั่งแอดมิน
   - Dashboard แสดงจำนวนตั๋ว แท็ก สถานะ
   - ดูตั๋วทั้งหมด + ฟิลเตอร์ (สถานะ, หมวดหมู่, ผู้ใช้)
   - ตอบกลับตั๋ว
   - เปลี่ยนสถานะ (เปิด / กำลังดำเนินการ / ปิด)
   - ลบตั๋วหรือปิดตั๋ว

4. UI/UX
   - ใช้ฟอนต์ Anuphan (Google Fonts)
   - โทนสีหลัก:  
       primary: #2196F3  
       primary-light: #E3F2FD  
       primary-dark: #1976D2  
       neutral-gray: #546E7A  
   - Layout กว้าง 1200px กลางหน้าจอ
   - ปุ่มโค้งมน สไตล์ material-light
   - Card + Shadow เบาๆ
   - ใช้ Breadcrumb และ Badge บอกสถานะ

5. ความปลอดภัย
   - CSRF protection  
   - File validation  
   - ตรวจนามสกุลไฟล์  
   - จำกัดสิทธิ์ user/admin  
   - ป้องกัน SQL Injection  

6. โครงสร้างไฟล์ (ใช้ PHP)
   / (main folder)
   /src  
   /views  
   /controllers  
   /uploads (เก็บไฟล์โลคอล)  
   /config  

7. ภาษาที่ใช้: ไทยทั้งหมด
   - ป้ายกำกับ
   - ข้อความแจ้งเตือน
   - ป้ายสถานะ เช่น “รอดำเนินการ”, “กำลังดำเนินการ”, “ปิดตั๋วแล้ว”

สุดท้าย โปรดสร้าง UI / Code / API / Database Schema ให้พร้อมใช้งานได้จริง
path เรียบสวย โดยใช้ .htaccess
